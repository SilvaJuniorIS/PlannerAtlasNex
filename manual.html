Manual de uso do AtlasNex Action Planner

Este guia mostra como transformar o planner em um painel diário de execução: você planeja ações, acompanha o calendário, mede avanço, organiza o Kanban e mantém backup dos dados enquanto constrói a meta de R$ 10.000/mês.

1. Primeiros passos

Ao abrir o planner pela primeira vez, ele carrega tarefas iniciais ligadas à estratégia AtlasNex. Essas tarefas são salvas no navegador. Isso significa que você pode usar o sistema sem login, sem servidor e sem banco de dados obrigatório.

1
Abra o painel principal

Acesse a página inicial, confirme os KPIs no topo e veja o mês atual no calendário.

2
Revise as tarefas iniciais

Use o checklist para ajustar datas, categorias e valores conforme sua realidade.

3
Execute diariamente

Marque conclusões, adicione oportunidades e acompanhe a receita planejada.

Regra prática: se uma ação ajuda prospecção, produto, entrega, conteúdo, financeiro, marca AtlasNex ou SaaS, ela deve virar tarefa.

2. Como trabalhar com tarefas

Cada tarefa guarda título, data, categoria, prioridade, valor previsto, observações, status de conclusão e etapa Kanban. O objetivo é ligar execução operacional com resultado financeiro.

Ação rápida

Use o painel lateral para registrar tarefas simples. Ao salvar, a tarefa aparece no calendário, checklist e Kanban.

Modal completo

Clique em “Nova tarefa” ou em um dia do calendário para preencher observações e escolher uma etapa Kanban.

4. Indicadores e leitura financeira

Os cards do topo atualizam automaticamente quando você cria, edita, conclui ou exclui tarefas.

Meta mensal

Referência fixa de R$ 10.000/mês para orientar volume e qualidade das ações.

Concluídas

Mostra a proporção de tarefas feitas e ajuda a acompanhar consistência semanal.

Receita planejada

Soma valores previstos e separa receita concluída da receita em aberto.

Agora

Indica a fase estratégica atual: fundação comercial, oferta e primeiros clientes.

Atrasadas

Conta tarefas abertas com data anterior a hoje. Revise essas tarefas no início do dia.

Tarefas da semana

Ajuda a medir se a semana tem volume suficiente para gerar oportunidades reais.

6. Backup, restauração e segurança

Nesta versão, os dados ficam no navegador usando localStorage. Isso deixa o app simples, mas exige backup manual caso você troque de computador, limpe o navegador ou use outro dispositivo.

1
Exportar JSON

Clique em “Exportar JSON” para baixar uma cópia das tarefas. Faça isso ao menos uma vez por semana.

2
Importar JSON

Use “Importar JSON” para restaurar um backup no mesmo navegador ou em outro computador.

3
Restaurar iniciais

Use essa opção para voltar ao conjunto base de tarefas estratégicas. Ela substitui os dados atuais.

Importante: antes de restaurar tarefas iniciais ou limpar dados do navegador, exporte um backup JSON.

7. Caminho futuro para nuvem

A arquitetura atual separa armazenamento, tarefas, calendário e inicialização em módulos. Isso facilita evoluir para autenticação e sincronização com Supabase sem reescrever toda a interface.

O que muda no futuro

O localStorage pode continuar como cache local, enquanto uma tabela `tasks` no Supabase guarda tarefas por usuário.

O que permanece

A experiência de calendário, checklist, Kanban, filtros, KPIs e backup pode permanecer igual.
